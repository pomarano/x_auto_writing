# X 投稿エージェント — 仕様（半自動）

## 目的

**内省テック** ブランド向けに、X（旧 Twitter）用の投稿文案を **1日1本** 生成する。  
テーマは **仏教に関する一言（教え・概念）** と **日常生活での実践** のセット。

| 担当 | 作業 |
|------|------|
| **エージェント** | 題材調査、文案作成、下書きファイル保存、重複チェック |
| **ぽまらの** | 内容確認・修正、X へ投稿、投稿済みマーク |

投稿 API の自動化は **行わない**（規約・品質・炎上リスクのため）。

---

## 1投稿の構成（固定フォーマット）

```
【一言】仏教の教え・概念（1〜2文、140字以内目安）
【今日の一歩】今日できる具体的な実践（1〜2文）
【出典メモ】エージェント用。調査した経典・論疏・概説の参照（投稿本文には含めない）
```

### トーン

- 説教・断定ではなく、「〜と言われる」「〜という見方がある」
- 医療・診断の代替にならない（セルフケア・内省の範囲）
- CBT・ACT と矛盾しない（執着＝悪ではない、気づきと距離）
- ぽまらのの一人称は **使わない**（ブランドアカウント想定なら「です・ます」で中立）

### 文字数

- **投稿本文**: 日本語 **200〜260字** 目安（X の読みやすさ。長くても 280字以内）
- ハッシュタグは **0〜2個**（毎回同じにしない）。例: `#内省テック` `#執着を手放す`
- ブログ URL は **週1回程度** のみ（[執着記事](https://pomarano.site/usual/291/) など）

## 題材ローテーション（7日）

| 曜日 | 軸 | 仏教の切り口（例） |
|------|-----|-------------------|
| 月 | 気づく | 正念、気づき、苦の認識 |
| 火 | 思考との距離 | 空、無我、執着（upādāna） |
| 水 | 小さな行動 | 八正道の正業・正命、歩むこと |
| 木 | 人・関係 | 慈悲（karuṇā）、慈（mettā） |
| 金 | 物・コントロール | 無常、諸行無常、手放す |
| 土 | 身体・呼吸 | 安般守意、身体の感覚 |
| 日 | ふりかえり | 中道、過度でも不足でもない |

※ 曜日は目安。直近30日の下書きと被らない題材を優先。

## エージェントの調査ルール

1. **偽の「仏陀の言葉」禁止** — 出典不明の名言ツイート風文案は使わない  
2. **参照の優先順**  
   - 経典名・論点が確認できる概説（日本語の信頼できる百科・大学オープンコース・既知の論疏紹介）  
   - 概念説明は「原始仏教」「大乗」など系統を一言添える  
3. **現代への橋渡し** — 必ず「今日の一歩」で職場・SNS・家事などに落とす  
4. **重複回避** — `social/x-drafts/` の過去30件を読み、同じ概念・同じ実践例を避ける  
5. **センシティブ** — 自傷・他害・特定宗教の優劣・政治は扱わない

## 毎日の運用フロー

```
[朝 or 決まった時刻]
  エージェント起動（下記プロンプト）
       ↓
  social/x-drafts/今日の日付.md ができる
       ↓
  ぽまらのが確認（必要なら手直し）
       ↓
  X にコピペ投稿
       ↓
  status: posted に更新
```

### エージェントの起動方法

1. **自動（推奨）— launchd 1日1回**  
   - セットアップ: `.company/secretary/automation/x-daily/README.md`  
   - 毎朝 7:00 JST に Cursor SDK で下書き生成  
   - Mac が起動していれば Cursor IDE は不要

2. **手動（Cursor IDE）**  
   - `x-shuuchaku-agent-prompt.md` を Agent に貼る

3. **Loop（IDE を開きっぱなしにするときのみ）**  
   - `/loop 1d 今日のX下書きを x-shuuchaku-agent-spec に従って作成`
